# Workflows v1.0 Baseline

**Versão:** v1.0.0 (Lançado 16 Outubro 2025)

**Status:** Totalmente funcional

**Features:** Gerenciamento básico de eventos, avisos de conflito

## Workflow 1: Setup Inicial

**Input do Usuário:**
```bash
cd timeblock-organizer/cli
python -m venv venv
source venv/bin/activate
pip install -e .
timeblock init
```

**Output do Sistema:**
```terminal
Database created successfully!
Location: /home/user/timeblock-organizer/cli/data/timeblock.db
Ready to track your time blocks.
```

## Workflow 2: Adicionar Eventos Básicos

### Cenário 2.1: Evento Simples
```bash
timeblock add "Reunião time" -s 09:00 -e 10:30
```

**Output:**
```terminal
[OK] Event created successfully!
ID: 1
Reunião time
09:00 -> 10:30 (1.5h)
```

### Cenário 2.2: Formato Brasileiro
```bash
timeblock add "Almoço" -s 12h -e 13h
```

**Output:**
```terminal
[OK] Event created successfully!
ID: 2
Almoço
12:00 -> 13:00 (1.0h)
```

## Workflow 3: Eventos Cruzando Meia-Noite
```bash
timeblock add "Plantão noturno" -s 23h -e 2h
```

**Output:**
```terminal
[AVISO] Evento cruza meia-noite (termina dia seguinte)
[OK] Event created successfully!
ID: 4
Plantão noturno
23:00 -> 02:00 (3.0h)
Duração: 3.0h (termina em 2025-10-30)
```

## Workflow 4: Detecção de Conflitos
```bash
timeblock add "Meeting A" -s 14h -e 15h
timeblock add "Meeting B" -s 14h30 -e 16h
```

**Output:**
```terminal
[OK] Event created successfully!
ID: 5
Meeting A
14:00 -> 15:00 (1.0h)

[AVISO] Aviso: Conflita com evento "Meeting A"
  Sobreposição: 14:30-15:00 (30 minutos)
[OK] Event created successfully!
ID: 6
Meeting B
14:30 -> 16:00 (1.5h)
```

**Ponto-chave:** Sistema AVISA mas NÃO bloqueia.

timeblock list
